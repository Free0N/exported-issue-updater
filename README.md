Утилита обработки экспортируемых данных из Jira
===

Утилита, основываясь на конфигурации, получает из Jira данные о задачах
в форматах JSON и XML. Данные преобразуются в общую структуру после чего
передаются на обработку объектам-обработчикам.

Объекты-обработчики реализуются в виде groovy-классов, и с помощью
предоставляемого инструментария могут изменять информацию о различных
частях экспортированных данных (проектах, задачах, комментариях и т.п.).

Т.к. не все данные о задачах можно получить используя один URL, то информация собирается через 2 разных эндпоинта.
Первый - отдающий информацию о задачах в JSON-формате. Второй - в XML-формате. Обработанные данные с каждого
эндпоинта преобразуются в соответствующие dto и в консолидированом виде передаются в обработчики
(см. интерфейс `org.samearch.jira.util.migration.api.IssueDataFetcher` и его реализации)

Пример конфигурационного файла - issueUpdateConfig.yaml.example

Использование
===

1. Скопируйте пример конфигурационного файла `issueUpdateConfig.yaml.example`
   в `issueUpdateConfig.yaml` и отредактируйте его. Обязательными параметрами
   являются те, которые отвечают за получение данных (секция `dataFetchConfig`).
2. Создайте один или несколько обработчиков данных и поместите их в поддиректорию
   `updaters` текущей директории.
3. Запустите получение и обработку данных из Jira:
   ```shell
   java -jar jira-exported-issue-updater.jar --updatedExportedIssuesFilePath=./issues.json
   ```
   Если вы переименовали конфигурационный файл, то укажите путь к нему с помощью параметра
   `--updateConfiguration`:
   ```shell
   java -jar jira-exported-issue-updater.jar --updateExportedIssuesFilePath=./issues.json --updateConfiguration=newConfigurationFilePath.yaml
   ```
   Обработанные данные задач будут записаны в указанный файл (`issues.json`)
4. Импортируйте файл `issues.json` в Jira из раздела System -> Import and export -> External System Import выбрав для
   импорта вайриант `JSON`


Дополнительная информация (требования и т.п.)
===

Для работы утилиты требуется JVM версии 21+

Утилита включает в себя следующие библиотеки:
* Spring Context 6.1.14
* Groovy 4.0.26 (подключен как groovy-all:4.0.26:pom)
* Jackson Dataformat Yaml 2.18.2
* Slf4J Simple 2.0.16
  
  доп. настройки Slf4J Simple:
  - org.slf4j.simpleLogger.showDateTime=true
  - org.slf4j.simpleLogger.dateTimeFormat=yyyy-MM-dd HH:mm:ss.SSS
  - org.slf4j.simpleLogger.showThreadName=false


Обработчик экспортированных данных
===

Обработчик экспортированных данных должен реализовать один из интерфейсов пакета `org.samearch.jira.util.migration.api.updater`:
* AttachmentUpdater
* CommentUpdater
* CustomFieldUpdater
* IssueUpdater
* ProjectUpdater
* WorklogUpdater

Если для работы обработчика требуются дополнительные классы, то их инстансы могут передаваться в конструктор обработчика.
Такие классы должны помечаться аннотацией `org.springframework.stereotype.Component`. Иначе они не будут загружены
в контекст Spring.

Полезные бины для обработчика:
* `org.samearch.jira.util.migration.api.config.UpdateConfig` - представление конфигурационного yaml-файла

Ошибки при импорте в целевую систему
===

В случае генерирования информации как о задачах, так и о линках при импорте в целевую систему могут возникать ошибки
Сообщение об ошибке похоже на следующее:
```text
2025-06-30 01:13:47,034 ERROR - Error importing issue [externalId='1234', summary='null']
com.atlassian.jira.plugins.importer.external.ExternalException: Unable to create or update issue: [externalId='1234', summary='null']
    ...
```
Подобные ошибки связаны с тем, что в наборе импортируемых данных отустствует информация о слинкованной задаче и ранее
она не импортировалась в целевую систему.

ВНИМАНИЕ!
===

В конфигурации генерирования данных можно задать генерирование данных задач и одновременное включение информации о линках
между задачами. В этом случае при импорте данных на целевую систему Jira будет вести себя следующим образом с
пользовательскими полями: при импортировании данных пользовательского поля для него будет создан новый контекст, в который
будут включены все проекты, на задачи которых имеются линки. Если в системе уже существует поле с соответствующим именем
и контекстом, включающим все проекты из импортируемого файла, то данные будут записаны в него.
Для предсказуемой работы импорта лучше использовать раздельный импорт данных задач и линков между ними (т.е. дважды запускать
экспорт с разным конфигом)
